The Known vs. Unknown Gap
What public data can verify — and where outreach fills the rest
~78% Verifiable via Public Data
~22%
Layer 1 (Public): NPI Registry, PubMed, State Medical Boards — locate and verify current practice, specialty, credentials. No consent required for public FOIA data.
Layer 2 (Voluntary): The remaining ~22% require alumni outreach. These may be in non-clinical roles, early career, or have changed fields. The relationship engine turns "located" into "engaged."

Data Sources — Full Transparency
TIER 1 — PUBLIC, AUTOMATED
NPI Registry (NPPES) — FOIA-released, monthly updates. PubMed — daily publication index. Free, no consent needed.
TIER 2 — PUBLIC, CAREFUL USE
State Medical Boards — individual license verification OK. No bulk scraping. Confirms practice status.
TIER 3 — VOLUNTARY ONLY
Doximity, LinkedIn — alumni self-link only. Terms of service prohibit scraping. Never positioned as a data source.

  • Mentorship matching: signal-based, specialty-specific

The Loop

Days 91+
  • Every alumni response feeds back into the system
  • The engine learns: better timing, better matches
  • Institutional memory that compounds — nothing falls through
  • Full alumni lifecycle: student → resident → mentor → advocate
  • The continuum Lisa described — actually working at scale
The Principle
Each phase earns the next. If Phase 1 doesn't deliver value, you stop — no risk, no sunk cost. The university owns everything that's built. No vendor lock-in. No subscription. The system grows with COMP's needs, not around a product's limitations.
