Bugs Found & Fixed

Bug 1: Add Note Button Does Nothing Fixed

Symptom: Clicking "Add Note" on a profile appeared to do nothing.

Root cause: The note form opened in Section 5 of the profile (notes/timeline), which is often offscreen. The form rendered correctly but the user couldn't see it.

Fix: After toggling the note form open, requestAnimationFrame waits for DOM render, then scrollIntoView({ behavior: 'smooth', block: 'center' }) scrolls to the textarea, and focus() puts the cursor in it.

File: js/views/profile.js — event handler for data-action="start-note"

Bug 2: "Showing 50 of 168" With No Pagination Fixed

Symptom: Projects Alumni tab showed "Showing 50 of 168" but provided no way to see the remaining 118 alumni.

Root cause: Hard-capped at 50 with .slice(0, 50) and no pagination or load-more mechanism.

Fix: Added module-scoped alumniPageSize (starts at 50), a "Load More" button that increments by 50, and correct remaining-count display. Button reads "Show N more" where N = Math.min(50, remaining).

File: js/views/projects.js — alumniPageSize variable, Load More rendering + event wiring

Bug 3: Jan 1 Publication Dates / Misleading "Last Connected" Fixed

Symptom: All publications showed "Jan 1, 2026" timestamps. "Last touchpoint" dates were misleading because they included automated NPI scrapes and PubMed matches — not real human connections.

Root cause: PubMed data has year-only precision, defaulting to Jan 1. getLastTouchpoint() treats all touchpoint types equally — automated data matches and real outreach get mixed together.

Fix: Created getLastConnection() helper that filters to human connections only: source='manual' or source='outreach_engine', or type='outreach'|'note'|'meeting'. Updated all views to show "Connected: [date]" or "No connection yet" instead of generic touchpoint dates.

Files: js/utils/helpers.js, js/views/profile.js, js/views/directory.js, js/views/projects.js
