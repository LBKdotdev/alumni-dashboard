Decisions Made

Outreach sends = auto-invite

When you draft and send outreach from a project, the alumni automatically moves to "invited" status. This prevents the double-action problem — no need to send an email AND manually mark them invited. One action, two records. If the alumni was already confirmed or declined, the status is preserved (only sets to "invited" if no status existed).

CSS classes over inline styles for responsive

Moved project layout from inline flex styles to named CSS classes. Inline styles can't be overridden by media queries — you'd need JS to detect viewport width. CSS classes let the mobile breakpoint cleanly stack layouts without touching JS. The class names are semantic: .proj-card-layout, .proj-alumni-meta.

Status dropdown inline, not in a modal

Invite status is a tiny select dropdown right on the card — no modal, no separate panel. When you're scanning 50+ alumni, the last thing you want is a popup for each one. Change the dropdown, it saves instantly. Batch workflow: scroll through the list, flipping statuses as you go.
