Decisions Made

State filter added to filterAlumni()

The existing filter engine had campus, specialty, engagement, tags, and class year — but no state/location filter. Added filters.state support so project alumni lists can filter by practice state. This also benefits the directory if a state filter dropdown is added later.

Projects loaded with graceful fallback

projects.json fetch uses .catch(() => ({ json: () => [] })) so the app works even if the file is missing or the API doesn't return projects. Zero-project state renders a clean empty view.

Project cards compute alumni count live

Rather than storing a count in the project data, the card calls filterAlumni(alumni, project.filter, '') every render. This means the count updates automatically as alumni data changes — no stale numbers.
