Decisions Made

All tabs shipped with the shell

The detail shell and all 4 tab contents were built as one cohesive unit. Splitting them into 5 separate commits would have created artificial intermediate states where tabs existed but had no content. The board items mapped the work scope correctly — the implementation was just more efficient to deliver together.

Module-scoped tab state, not global state

The active tab is tracked in a module variable, not in state.js. This prevents unnecessary re-renders of the entire app when switching tabs. Tab resets to "alumni" when opening a project.

Alumni list capped at 50

The Vegas project has 168 alumni. Rendering all 168 as cards would slow the UI. Capped at 50 with a "Showing 50 of 168" note. Full pagination comes in Board 4.
