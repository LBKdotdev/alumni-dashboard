What

Phase 1 — Deploy
Get it live on a URL Lisa can hit from her phone
  • Cloudflare Pages deploy — connect the private GitHub repo, deploy to a .pages.dev URL. No build step needed, static files only. Free tier.
  • Cloudflare Access — gate the site behind email authentication. Only Lisa's email (and yours, and Joey's) can access. Everyone else sees a login wall. Free for up to 50 users.
  • Custom domain (optional) — nacilink.dev or similar. Or just use the .pages.dev URL for now.
  • HTTPS — automatic with Cloudflare Pages. Zero config.
Phase 2 — Conference-Ready Features
Small additions that make a big difference at AACOM
  • Quick lookup mode — one-tap search from login. Lisa opens NaciLink, types a name, gets the profile. Minimum friction.
  • Vegas filter pre-set — one-tap filter to show only the 168 Nevada alumni. "Who's local to this conference?"
  • "Met at AACOM" tag — quick-add tag Lisa taps after meeting someone. Post-conference, she filters by this tag and has her follow-up list.
  • Quick note on mobile — after a conversation, Lisa adds a note in 10 seconds. "Interested in preceptor role. Follow up re: spring visit." Done before she forgets.
  • Add to Home Screen — PWA manifest so Lisa can install NaciLink as an app icon on her iPhone. Feels native, launches full-screen, no browser chrome.
