NaciLink Conference Features

Front-end sprint — everything Scotty can ship while Joey builds the D1 backend.

Plan — Awaiting Approval

Intent

Build every front-end feature that makes NaciLink a conference weapon. All UI, all state logic, all ready to wire to Joey's API endpoints when they land. Zero throwaway work — every line of code ships to production.

Purpose

Lisa walks into AACOM Vegas with NaciLink on her phone feeling like a native app. She can search alumni in 2 seconds, tag people she meets, filter the Vegas list, and look up anyone before a conversation. The front-end is the experience. The backend is the persistence. Scotty owns the experience.

Scope

1. PWA Manifest — Home Screen Install 3 pts

Add manifest.json with NaciLink name, icons (circular emblem at 192px and 512px), theme color (#111111), background color (#111111), display: standalone. Add meta tags to index.html: link rel="manifest", meta name="theme-color", Apple touch icon. Lisa taps "Add to Home Screen" on Safari — NaciLink icon on her iPhone, launches full-screen, no browser chrome, no URL bar. Feels native.

2. Vegas Filter Preset 3 pts

Add an "AACOM Vegas" chip/button to the Directory view filter bar. One tap filters to practice_state = NV. Shows "168 alumni in Nevada" count. Second tap clears the filter. Styled as a teal pill button with a location pin icon. Visible on mobile without scrolling — sits above the alumni list, below the search bar. Lisa opens Directory, taps Vegas, sees her conference list.

3. "Met at AACOM" Quick Tag 5 pts

Add a floating action button on the profile view — teal circle, handshake or check icon. One tap adds "met-at-aacom" to the alumni's tags array and adds a touchpoint: "Met at AACOM 2026 — tagged by Dr. Warren." Visual confirmation: button turns green with checkmark, "Tagged" text. If already tagged, button shows as green (idempotent). On Directory view, add "Met at AACOM" as a filter option — Lisa filters post-conference to see everyone she connected with. This is the follow-up list generator.

4. Conference Quick Lookup 5 pts

New view: full-screen search optimized for standing in a crowd. Big search input (auto-focused), large result cards (name, specialty, city — nothing else). Tap a result → full profile slides in. Back button returns to search with results preserved. No sidebar, no nav clutter, no dashboard — just search and go. Accessible from a dedicated icon on mobile nav or a gesture. Think iPhone Spotlight: pull down, type, find, done. This is how Lisa uses NaciLink mid-conversation without fumbling through menus.

5. NaciLink Branding Pass 3 pts

Wire the NaciLink logo into the login gate (already started — finalize sizing and spacing). Update sidebar brand from "Alumni Engine" to "NaciLink" with the circular emblem. Update mobile nav header. Update page title in browser tab. Update the Be reports to reference NaciLink. The tool has a name now — make sure it shows up everywhere consistently.

6. Alumni-to-Alumni Connection Assist 3 pts

On a profile view, add a "Similar Alumni" section at the bottom — 3 alumni who share the same specialty in the same state or nearby city. Lisa meets Dr. A in cardiology, scrolls down, sees 3 other cardiologists nearby. "Let me connect you with Dr. B — she's 20 minutes from your practice." Lisa becomes a connector, not just a contact collector. Simple matching: same specialty + same state, sorted by city proximity to the viewed alumni.

Approach

  1. PWA manifest + home screen install (foundation — do first)
  2. NaciLink branding pass (visual identity locked in)
  3. Vegas filter preset (quick win, immediately useful)
  4. "Met at AACOM" quick tag (the conference killer feature)
  5. Conference quick lookup (the UX differentiator)
  6. Alumni-to-alumni connection assist (the "wow" moment)

Order is by dependency then impact. PWA and branding first because they're the container everything lives in. Vegas filter and AACOM tag next because they're the highest-value conference features. Quick lookup and connection assist last because they're the most complex but deliver the biggest "wow."

Constraints

  • No persistence until Joey ships D1. Tags and notes added at the conference will live in browser state only. When D1 is ready, wire the state mutations to API calls. The front-end code doesn't change — only the persistence layer underneath it.
  • State module is the integration point. All mutations go through state.js. When D1 lands, each mutation function gets an await fetch('/api/...') call added. The views and components never know the difference.
  • No new dependencies. Vanilla JS. No service workers for offline (that's Phase 2). No frameworks. No build tools.
