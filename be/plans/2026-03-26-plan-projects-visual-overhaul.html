Projects Tab + Visual Overhaul

Add a fourth nav tab for proactive work (campaigns, events, lists) and fix the visual hierarchy so nothing washes out.

Plan — Awaiting Approval

Intent

Dr. Warren doesn't just react to triggers — she runs campaigns. Vegas dinner, reunion planning, SOAP mentor recruitment, donor drives. The app has no place for this proactive work. Everything is either reactive (Queue) or passive (Directory, Dashboard).

Meanwhile, the visual design bleeds details into the background. Gray on gray on cream. Metadata, insights, and context — the things that make this tool smart — are the hardest things to read.

This plan fixes both. Add the missing workflow (Projects) and make the whole app scannable at a glance.

Scope

Projects Tab — What It Is

A fourth navigation item between Directory and Dashboard. Contains project cards — each scoped to an event, campaign, or initiative.

Project Data Model

projects.json

Each project stores:

  • name — "AACOM Vegas 2026"
  • type — conference | reunion | campaign | initiative
  • date_range — start/end dates (for countdown)
  • filter — alumni filter criteria (state, class_year, specialty, tags)
  • checklist — array of {task, done, due_date}
  • outreach_template — pre-drafted email for this project
  • brief — markdown intel brief (talking points, stats)
  • alumni_status — per-alumni tracking: invited / confirmed / declined / no-response

Approach

Phase 1: Projects Foundation
Nav item, state management, projects.json data structure, landing view with project cards. Seed with AACOM Vegas project.
Phase 2: Project Detail View
Inside-a-project view with tabbed sections: Alumni list (filtered from directory data), Checklist, Outreach templates, Intel brief. Alumni list reuses directory filter/sort engine.
Phase 3: Visual Hierarchy
Trigger card tinting, profile section accents, queue header redesign, insight text strengthening. Touch every view to ensure scanability.
Phase 4: Per-Alumni Project Tracking
Inside a project, track invite status per alumni (invited/confirmed/declined). Show status badges on alumni cards. Outreach sent from project gets recorded on the alumni's touchpoint timeline.
Phase 5: Mobile Polish + QA
Projects view responsive on phone. Project detail tabs work via swipe/tap. Test on iPhone Safari, Android Chrome. Full QA pass on all views.

Constraints

  • No frameworks. Vanilla HTML/CSS/JS only. Same architecture as existing views.
  • No new dependencies. No build tools, no npm packages.
  • projects.json is static for now. Edits are in-memory (like notes/outreach). Persists to file when Workers + KV go live.
  • Reuse directory filter engine. Project alumni lists use the same filterAlumni/sortAlumni functions. No duplicated logic.
  • AACOM Vegas is the test case. Every feature ships working against the real Vegas data. Not hypothetical.
  • Mobile-first for project detail. Dr. Warren will use this at dinner on her phone. It must work at 375px.
  • 27 days to Vegas. Phase 1-3 must ship before April 18 (her test window).
